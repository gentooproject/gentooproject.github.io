El acortador de links más privado del mundo

1. El sitio funciona con github, por lo que el uso que hagas del sitio permanece en secreto para todos, incluyendo a los creadores del sitio (@princessgentoo). La información del uso del sitio la usa github para mejorar sus servicios y asegura que esa información no se usa para identificar a nadie.

También el sitio web de Qubes OS, el cual se perfila como uno de los sistemas operativos más seguros que existen y que utiliza Snowden. En lo personal uso y también puedo dar constancia de él.¿Por que usamos Cloudflare? Porque el mantener un servidor es bastante costoso,Cloudflare es una solución para las personas que no podemos invertir gran cantidad de tiempo, dinero y recursos humanos.

3. Ni siquiera Github ni Cloudflare sabe que estás visitando un link acortado, porque el link se genera en tu PC.

4. Es el primer acortador de links 100% transparente, pues los sitios con Github muestran públicamente el código con el que funciona todo el sitio: De esta forma se puede descargar el sitio entero ejecutando:

git clone github.com/gentooproject/gentooproject.github.git

Y puedes explorar y descargar sus archivos en:

https://github.com/gentooproject/gentooproject.github.io

Es importante destacar que github lleva un registro estricto de cualquier cambio y un historial de versiones en tiempo real. Por lo que cualquier mínimo cambio aparecerá registrado y podrás consultarlo en el link de arriba.

5. rpm.li no aloja ningún contenido ni utiliza ninguna base de datos. Por un lado porque una base de datos puede ser comprometida, y segundo porque mantener una base de datos sería vulnerar la privacidad de los que usen el servicio al mantener una lista de los sitios que se visitan y registran. Lo único que aloja rpm.li es un javascript

6. Dependiendo de la url ingresada, por ejemplo: rpm.li/123, donde 123 es el id, el javascript carga por medio de esta id el contenido de un sitio de pastebin.

Sólo los miembros con cuenta pro o premium pueden subir hiperlinks a pastebin, los usuarios anónimos y standard sólo pueden subir links planos.

Consejos para aumentar la seguridad:

  • Para evitar leaks de DNS y doble consulta de DNS, puedes agregar rpm.li a /etc/resolv.conf y así evitar un cambio no autorizado de servidor (como que deje de usar Github)
  • Edita el HTTP referer en tu explorador web para que los sitios que visites no sepan que los visitas de pastebin o rpm.li

Por las mismas razones que el sitio de Qubes OS (El sistema operativo que usa Snowden) está alojado en Github como lo definieron ellos, es que no puedes confiar en los puntos medios como lo es el server. Incluso un server que funciona con la distribución de Richard Stallman y alojado en Islandia no garantizan la privacidad si es ejecutado por las manos equivocadas. Por tanto es más confiable Github que un server con las carecterísticas mencionadas allá arriba. Si de verdad deseas privacidad implementala tu mismo (backend) ya sea con VPN, tor, etc...

Por mantener prioridad sobre la privacidad en vez de la seguridad como se explicó con Cloudflare, el sitio no puede ser servido en https, sin embargo, el contenido de los links acortados si cargan utilizando HTTPS. No obstante, solucionaré este problema en el futuro creando una extensión para Chrome y Firefox que te permita ejecutar rpm.li offline y de forma más segura.
