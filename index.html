Este sitio, así como el de Qubes OS y el de Snowden freedom.press utiliza Cloudflare, el cual tiene acceso a información confidencial como tu IP, tu navegador web, tu sistema operativo. .¿Por que usarlo? Porque el mantener un servidor es bastante costoso, Cloudflare es una solución para las personas que no podemos invertir gran cantidad de tiempo, dinero y recursos humanos. Nos permite implementar de manera rápida, y económica soluciones de seguridad de alta tecnología como HTST.

  • Cloudflare ofrece HTTPS gratuito para cualquier sitio, entonces un internet sin HTTPS ya no es justificable a pesar de que la mayoría de las páginas web no utilicen HTTPS
  • Gestiona los certificados de HTTPS de forma automática y de esta forma, la página nunca dejar de funcionar ni tendrá que enfrentarse a certificados expirados.
  • Te permite redireccionar automáticamente todo tu contenido a HTTPS, pues configurar manualmente la redirección de todo el sitio a https puede volverse algo trival o estar restringido por el propio hosting.
  • Ofrece certificados comodín HTTPS para todos los subdominios, que si bien esto ya es un servicio de pago, son más accecibles que comprarlo con otras compañías.
  • Permite implementar HTST, que es un protocolo de alta seguridad que busca solucionar las debilidades de TLS (HTTPS), el cual de otra forma para la mayoría de los servidores no podra implementarse.
  • También otra tecnología que permite utilizar Cloudflare es DNSSEC que garantiza que la resolución de nombres de dominio como rpm.li, google.com no sea vulnerada.

Estas tecnologías son el standard para sitios como google.com, pero están disponibles a todo el público en general gracias a Cloudflare. No se busca dar publicidad a Cloudflare, y es por eso que se hará una denuncia y crítica a este servicio.

Todas estas tecnologías están implementadas en rpm.li, en excepción de DNSSEC, tecnología que se busca implementar como uno de los objetivos de este sitio. Se está contemplando utilizar un certificado SSL que utilice sólamente los protocolos de encriptado más actuales y poderosos dejando de dar soporte los encriptados menos seguros.

El gran revés de Cloudflare es el uso obligatorio de sus certificados privados; se ha sugerido fuertemente y hay estudios serios que avalan esta posibilidad, que la NSA es capaz de decifrar 2/3 de los encriptados TLS y VPN utilizando una teoría matemática de números primos. Ante esta violación de los derechos humanos de la NSA lo ideal es poder utilizar los certificados gratuitos y libres de Let's Encrypt, pero para poder hacer esto en Cloudflare hay que convertirse en cliente Premium. La seguridad ideal más económica sería configurando un VPS o servidor dedicado. Pero implica dos cosas: Conocimiento (recursos humanos) y tiempo; el tiempo no es dinero.

Para liberarte de esta supercookie debes seleccionar borrar historial reciente en firefox, y seleccionar la última opción ; idealmente puedes seleccionar todas las opciones si deseas borrar todo tu historial.

4. Github ni Cloudflare saben que estás visitando un link acortado, porque el link se genera en tu PC.

5. Es el primer acortador de links 100% transparente, pues los sitios con Github muestran públicamente el código con el que funciona todo el sitio: De esta forma se puede descargar el sitio entero ejecutando:

git clone github.com/gentooproject/gentooproject.github.git

Y puedes explorar y descargar sus archivos en:

https://github.com/gentooproject/gentooproject.github.io

Es importante destacar que github lleva un registro estricto de cualquier cambio y un historial de versiones en tiempo real. Por lo que cualquier mínimo cambio aparecerá registrado y podrás consultarlo en el link de arriba.

6. rpm.li no aloja ningún dato ni utiliza ninguna base de datos. Por un lado porque una base de datos puede ser comprometida, y segundo porque mantener una base de datos sería vulnerar la privacidad de los que usen el servicio al mantener una lista de los sitios que se visitan y registran. Lo único que aloja rpm.li es un javascript
